# 채널·파트너십(RTM) — 비치헤드 도달은 "직접 인사이드세일즈 ×(행정사·노무사 임베드 + 산단·협회 세미나)"의 2-엔진 믹스로, 은행연계(D3)는 데이터 축적 후행 파트너십으로 단계화한다

## ① 핵심 질문
중소·중견 제조·요양(외국인 5~50명 고용) 비치헤드에 닿기 위한 **RTM·채널 믹스**(다이렉트 인/아웃바운드 · 행정사·노무사 제휴 · 산단/업종협회/외국인고용협회 · 정부·지자체 B2G · 송출입기관), 각 채널의 **경제성(CAC·도달규모·전환·리드타임)**, **파트너십 모델(레퍼럴·리셀러·임베드)**, 그리고 **D3 은행연계 파트너십**(케이뱅크·토스뱅크: 송객→스코어→공동여신, 규제 후행)을 어떻게 설계·우선순위화하는가.

## ② 분석

### 2.1 채널 도달 구조 — 비치헤드는 "조직되어 있다"
비치헤드 고객(외국인 5~50명 고용 중소·중견 제조·요양)의 결정적 특징은 **이미 두 개의 기존 채널에 묶여 있다**는 점이다.
- **(a) 전문가 채널** — 외국인 담당자가 비자·체류 처리를 행정사/노무사에게 외주(사후·간헐). PlugNX가 보유망(행정사 3,360·변호사 365 라우팅 — 출처: strategy.html, PROJECT.md 해자)을 갖는다.
- **(b) 집합 채널** — 사업장이 산업단지·업종협회·외국인고용협회(예: 중소기업중앙회 외국인력고용지원, 산업인력공단 EPS 권역)에 물리적/제도적으로 묶여 있다.

→ **함의: 다이렉트(콜드)로 영세 사업주를 한 명씩 뚫는 것은 CAC가 가장 비싸다.** 이미 신뢰를 쥔 (a)(b)에 **임베드/공동 세미나**로 올라타는 것이 80/20이다.

**판독**: C3(행정사·노무사) 26 > C4(협회 세미나) 24가 명백한 우상단. **PlugNX가 이미 보유한 라우팅망(3,360·365)을 "외주 사후처리"에서 "상시 유통 채널"로 전환**하는 것이 가장 빠르고 싸고 반복가능하다. C2(아웃바운드)는 C3/C4가 데우지 못한 잔여 수요를 직접 마감하는 보조 엔진.

### 2.3 권고 채널 믹스 — 2-엔진 + 1-수요창출
```
[엔진1] 직접 인사이드세일즈(C2)  ──┐
                                    ├─→ 유료 전환 마감(클로징·PoC 운영)
[엔진2] 행정사·노무사 임베드(C3)  ──┘   ↑
                                        리드 공급
[수요창출] 산단·협회·고용협회 세미나(C4) ── 다수 사업장 동시 접점 + B2B2C 노출
[후행] B2G(C5)·송출입(C6) = 신뢰·확장 자산   |   [자동] 근로자 Moneeback 유입(C7, CAC0)
```
- **세미나(C4)로 다수를 동시에 데우고 → 행정사·노무사(C3)가 자기 수임 고객을 PlugNX로 데려오며 → 인사이드세일즈(C2)가 PoC·유료를 마감**한다. PROJECT.md의 검증 깔때기("산단공단·상공회의소·요양협회 세미나 → 무료 PoC → 유료")와 정합.

### 2.5 D3 은행연계 파트너십 — 송객 → 스코어 → 공동여신 (규제 후행 단계화)
> 전제: **"대출=수익엔진"은 최종 도착지이지 1단계가 아니다.** VisaDesk(데이터 입구) → Moneeback(환급=CAC0 미끼·송객풀) → **데이터 축적 후** D3 여신(strategy.html·screening.html·business-case 10 '여신 후행'과 정합).

**왜 은행이 테이블에 앉는가**(screening.html 근거): 인터넷은행 외국인 신용대출 **0**(토스뱅크 외국인 계좌 21.4만·예치 2,816억이나 여신 0). 병목은 단 하나 — **외국인 신용평가 데이터 부재(씬파일러)**. PlugNX 고용·임금·근태·체류 데이터가 그 빈칸을 메운다.

**3단계 파트너십 로드맵**(규제난이도 순 — 출처: screening.html ③ 연계모델 3안):
| 단계 | 파트너십 형태 | 규제난이도 | 필요 등록/라이선스(참고용) | PlugNX 수익 | 시점(가정) |
|---|---|---|---|---|---|
| **A. 대출 송객·리드** | 송객 제휴(LOI→파일럿) | 낮음 | 온라인 대출모집(대출성 상품 중개) 등록 | 중개수수료 | 데이터 축적 후 우선 타진 |
| **B. 대안신용 스코어 공급** | 데이터·스코어 공급 계약 | 중 | 전문개인신용평가업(전문CB) 인허가 — 자본금 정형 20억/비정형 5억 등 | 조회·라이선스 + 성과연동 | 데이터 임계량 후 |
| **C. 공동여신/마이데이터** | 통합 금융 플랫폼 | 높음 | 본인신용정보관리업(마이데이터) 허가 | 통합 금융 플랫폼 수익 | 장기 |

**협상 레버**(PlugNX가 쥔 패 — screening.html ⑥): ① 유일 데이터 소스(은행이 못 만드는 씬파일러 스코어 원료) ② 송객 채널 CAC0(플라이휠 외국인 풀) ③ 정책 정합(체류 적법성 검증 → 대포통장·불법체류 리스크 사전 경감). **벤치마크**: Nova Credit "Credit Passport"(이민자 해외신용 이전, 20개국+·시리즈C 4,500만 달러) = 한국형 모델 원형.

> ⚠️ **금융·규제는 참고용·전문가 확인 필요.** 외국인등록번호(민감정보) 처리, 수집·이용·제3자 제공 동의, 국내보관·암호화, 신용정보법·개인정보보호법 적용범위, 라이선스 요건·자본금·수수료율은 모두 변동되며 법적 효력 없는 참고치 — 금융당국·변호사·컴플라이언스 사전 검토 필수.

### 2.6 B2B2C 자동유입 — 기업 채널 = CAC 0 근로자 풀 (송객 자산)
플라이휠(PROJECT.md): **기업이 근로자 등록 → 근로자 Moneeback 자동 유입(전환 60%+ 목표·가정, CAC 0) → 만족 → 기업 근속·유지.**
- **자동 유입 지점**(명시): ①기업 온보딩 시 근로자 일괄 등록 → Moneeback 환급 자격 알림(앱/6개 언어) ②급여·체류 데이터 이미 보유 → 환급 추정액 자동 제시 ③ForeignID L1~L3 인증이 두 서비스를 한 신원·데이터 레이어로 묶음(strategy.html).
- **채널 경제성 의미**: B2B 채널 1건 = B2C 근로자 5~50명 무료 유입. **이 풀이 곧 D3 은행 송객 채널(2.5)** — B2B 채널 투자가 D3 후행 수익의 송객 자산으로 복리화된다(CAC 0 연쇄).

## ③ So-what (전략적 함의)
1. **PlugNX의 진짜 채널 자산은 "신규 영업조직"이 아니라 이미 보유한 행정사·노무사 라우팅망이다.** 이를 사후 외주에서 상시 유통 채널로 전환하면 CAC를 구조적으로 낮추고 8주 PoC 도달을 가속한다.
2. **세미나(수요창출) → 파트너 레퍼럴(리드) → 인사이드세일즈(마감)의 깔때기**가 PROJECT.md 검증 깔때기와 1:1 정합 — 새 설계 불필요, 실행만 남았다.
3. **B2B 채널 1건이 B2C 근로자 풀(CAC0)을 만들고, 그 풀이 다시 D3 은행 송객 자산이 된다** — 채널 투자가 단계마다 복리로 재사용된다. D3는 무리하게 당기지 않고 데이터 축적 후 송객(A)부터 단계화.
4. **B2G·송출입은 즉시 매출 채널이 아니라 신뢰·레퍼런스·확장 자산**으로 포지셔닝(리드타임 길고 전환 불확실).

## ⑤ 의존성·오픈이슈
- **01 세그멘테이션**(파일 부재): 세그별(업종×규모×비자×지역) 우선순위가 확정되면 **채널 가중치·세미나 버티컬·파트너 선정 기준 재조정** 필요. 현재는 브리프 비치헤드(제조·요양·5~50명)로 **추정** 진행.
- **04 가치·가격**(파일 부재): 레퍼럴 커미션율(첫해 10~20% 가정)은 **04의 LTV·마진과 정합 확인 필요**. 온보딩비(30~100만 SSOT) 면제를 협회 회원혜택으로 쓰는 안의 마진 영향 확인.
- **06 영업모션**: 인사이드세일즈 퍼널 단계·전환율·인에이블먼트는 06이 정의 — **파트너 리드 등록→마감 핸드오프 SLA를 06과 정합** 필요.
- **08 비즈니스 케이스**: 채널별 CAC(여기 "가정")는 08의 유닛이코노믹스 입력 — **CAC 수치 정합/실측치 단일화** 필요.
- **07 런치·PoC**: 첫 파트너 5곳·세미나 2회·8주 KPI는 07의 30/60/90 일정과 **캘린더 정합** 필요.
- **확인 필요(외부)**: 행정사/노무사의 SaaS 리셀러·레퍼럴 수취가 직역 윤리·관련 법규상 허용 범위인지 **전문가 확인**. 대출모집 등록·전문CB·마이데이터 요건은 **금융당국·변호사 확인**(screening.html 라이선스 요건은 변동·참고치).

## ⑥ 헤드라인
보유한 행정사·노무사망(3,360·365)을 사후 외주에서 상시 유통 채널로 전환하고, 산단·협회 공동세미나로 수요를 데워 인사이드세일즈가 마감하는 2-엔진 믹스를 가동하라 — 은행연계(D3)는 B2B2C로 쌓은 CAC0 송객풀 위에서 데이터 축적 후 송객(모델 A)부터 후행 단계화한다.
