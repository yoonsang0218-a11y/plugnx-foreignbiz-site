## ① 핵심 질문

창업자의 의심을 정면으로 다룬다: **"기업들이 그냥 엑셀 쓰면 되는데, 굳이 돈 주고 컴플라이언스 기록을 만들 필요가 있을까?"**

1. 구독료는 사업 모델에서 무슨 역할인가 (내부 명제)? 고객은 무슨 명제에 돈을 내는가 (판매 명제)?
2. 고객이 실제로 지불하는 조건은 무엇이며(MECE 4조건), 조건이 없으면 누구에게 팔지 말아야 하나?
3. 엑셀·더존/ERP·행정사·노무사·무대응 각각을 어떻게 격파하나 — 고객의 반박과 재반박 포함?
4. 30명 사업장 기준 ROI 산수는 보수/기본 시나리오에서 어떻게 나오나 (정직한 버전)?
5. PoC 8주 안에서 가격·메시지의 무엇을 실측하고, 어떤 결과면 어느 길로 가나 (decision rule)?
6. "준법관리 SaaS" → "외국인 인력 유지 OS" 리포지셔닝의 카피는?

**한 문장 답 ("월 89만원을 왜 내는가")**:
> **외국인이 ~10명을 넘는 순간 엑셀의 진짜 가격(담당자 시간 + 누락 1건의 기대손실 + 퇴사 시 기록 증발)이 구독료를 넘어서고, 단속은 실제로 집행되며(’25 1차 합동단속 2,263명·101억), 여기에만 있는 환급 복지(Moneeback)가 이탈 1명(대체비용 약 1,500만·가정)을 막아주기 때문이다.** 단, 이 문장이 성립하지 않는 고객(외국인 5명 이하·단일 비자·이탈/단속 무경험)에게는 팔지 않는다.

## ② 분석

### A. "왜 돈 내는가" 논리 트리 (Pyramid)

#### A-0. 지배 명제 — 구독료의 역할 (내부 전략, 고객에게 말하지 않음)

```
[내부] 구독료의 진짜 역할 2가지
├── ① 진지함 필터: 돈을 낸 기업만 근로자 데이터를 성실히 올린다.
│      무료면 활성화가 무너지고(07 R1: "담당자가 엑셀 데이터를 안 올림"),
│      활성화가 무너지면 플라이휠(B2C 자동유입 60%+·가정)도 무너진다.
├── ② 데이터 입구 유지비: 구독료는 고용·임금 데이터 파이프라인의 입장권.
│      수익 본체는 데이터 위 금융(환급 성공보수 → 후행 신용·여신, strategy 시퀀스)
│      — vivisa×클링커즈 동맹(’25.12)이 같은 길목을 노리므로 데이터 선점은 속도전
│      (handoff/research-b2c-apps.md).
└── 따라서: 구독료가 ARR의 전부가 아니어도 "유료"는 포기 불가
       — 유료성 자체가 데이터 품질 담보 장치다.
```

> **고객에게 파는 명제는 별개다**: "외국인 인력을 잃지 않게 하는 운영 시스템(준법+복지)". 내부 명제(데이터 입구)를 영업 자리에서 말하면 신뢰를 잃는다.

#### A-1. 고객이 지불하는 4조건 (MECE) — 하나라도 강하면 WTP 발생, 전부 없으면 no-sell

```
고객 판매 명제: "외국인 인력을 안 잃는다 (적발로도, 이탈로도)"
├── 조건1. 집행되는 처벌 — 벌칙이 종이호랑이가 아님
├── 조건2. 엑셀의 임계점 — 인원·복잡도가 수기 한계를 넘음
├── 조건3. 제도 강제 — 기록이 평가·쿼터와 연결됨 (시행 시 점프)
└── 조건4. 이탈 방지 — 환급 복지로 근속↑ (우리만의 카드)
```

**조건 1 — 집행되는 처벌** (적발은 추상이 아니라 통계다)

| 사실 | 수치 | 출처·라벨 |
|---|---|---|
| ’25년 1차 합동단속(77일) 적발 불법고용주 | **2,263명** | 법무부 보도자료(’25.4.14~6.29)·언론 — 1차 출처 재확인 권장 |
| 동 기간 범칙금 합계 | **약 101억원** | 동상 |
| 1건당 평균 부과액 | **약 446만원** (101억÷2,263, 산술평균·분포 아님) | 산식 — **02 문서의 "4,460만"은 오류, 교정** |
| 불법고용 벌칙 상한 | **3년 이하 징역 또는 벌금 최대 3,000만원** (§94, ’26.1.23 상향 시행) | SSOT/브리프 — 위반 시점 따라 구법 적용 가능, 확인 필요 |
| 부수 제재 | 외국인 **고용제한(최대 3년)** 가능 | 외국인고용법 §20 — 요건·기간 확인 필요 |
| 단속 빈도 | 합동단속 **연 2회 정례화** 기조 | 정부 발표 기조 — **확인 필요**(1차 출처 미확정) |

→ **정직한 프레임**: "평균은 446만이지만, 상한은 3,000만 + 고용제한이다." 평균(빈도 리스크)이 구독료의 일상 정당화, 상한(꼬리 리스크)이 보험 프레임. 둘을 섞어 "평균 4천만"처럼 말하면 과장이고, 분리해서 말하면 진실이며 둘 다 영업에 쓸 수 있다.

**조건 2 — 엑셀의 임계점 (임계 N의 정식화)** 〔전체 가정·PoC 실측 대상〕

관리 부하를 변수로 정식화한다:

> **연간 추적 이벤트 T ≈ N × e**, 오류 위험은 **V(비자종류 수)·E(연간 제도변경)·R(담당자 교체)**로 가중.

| 변수 | 정의 | 추정값〔가정〕 |
|---|---|---|
| N | 외국인 근로자 수 | 비치헤드 5~50 |
| e | 1인당 연간 관리 이벤트 수 (체류만료 체크포인트 D-120/90/60/30, 갱신, 고용변동신고, 임금요건 확인, 보험, 계약) | **5~8건/인·년** |
| V | 비자 종류 수 (E-9·E-7·H-2 등 — 종류마다 규칙·기한 상이) | V≥2부터 오류율 비선형 증가 |
| E | 연간 외부 변경 이벤트 (임금요건 고시·쿼터 변경·법 개정 — 예: §94 상향, E-9 쿼터 16→13→8만) | 연 3~5건 |
| R | 담당자 교체 리스크 (겸직 1명 = 버스팩터 1) | 비치헤드 기본값 = 겸직 1명 |

**임계점 가설**〔가정〕: T가 **연 100건(주 2건+)을 넘고 V≥2이며 담당이 겸직**이면 엑셀+캘린더의 누락 확률이 관리 가능 수준을 벗어난다. T=100 ÷ e(5~8) → **임계 N\* ≈ 10~15명**.

| 구간 | 판정 | GTM 함의 |
|---|---|---|
| N ≤ 5, 단일 비자 | **엑셀이 이긴다** — 정직하게 인정 | no-sell(셀프서브·무료진단만) |
| N 5~10 | 경계 — 트리거(이탈·단속 경험) 있을 때만 | Starter 진입로, 사람 영업 금지(06 정합) |
| **N 10~50** | **시스템이 이긴다** — 임계 초과 | **유료 코어(Growth/Compliance)** — 비치헤드 5~50과 정합 |
| N 50+ 다사업장 | 시스템 필수이나 조달·커스텀 장벽 | Enterprise, PoC 단계 후순위 |

→ "5명까지는 엑셀이 이긴다"를 **우리 입으로 먼저 말하는 것**이 신뢰 자산이다. (메시지 F-5 참조)

**조건 3 — 제도 강제 (K-Trust 등)** 〔시행 전 WTP ≈ 0, 시행 시 점프〕

- 시행 전: 이 조건만으로는 돈을 내지 않는다 — 단정 영업 금지(02 proof 가드: 쿼터 가감폭 ±20~30%는 1차 출처 미확인).
- 시행 시: 평가·쿼터가 "평소 기록"을 보는 구조라면 **기록은 소급 생성이 불가** → "시행 후 시작하면 늦다"는 선점 논리가 성립.
- 영업 사용법: 주 무기가 아니라 **클로징 보조 화살**("준비는 소급이 안 됩니다" — 06 반론처리 정합). 시행 일정·세부는 항상 "확인 필요" 병기.

**조건 4 — 이탈 방지 (체감 ROI, 경쟁 대체재가 흉내 못 내는 우리만의 카드)**

| 요소 | 수치 | 라벨 |
|---|---|---|
| E-9 등 이탈/공백 1명의 대체 비용 (재채용·재교육·공백 생산성, ~3개월) | **약 1,500만원** | 〔가정〕(business-case 03 인용값, 산정식 미검증) |
| Moneeback이 찾아주는 미수령 환급 (연금 반환일시금 1인 평균 870만~1,094만 + 퇴직금·보험 합산) | **약 1,000만원대** | 국민연금공단·언론(’25.10) + 합산 〔추정〕 |
| 쿼터 환경 | E-9 ’26 쿼터 **8만**(16→13→8만 감소) — 이탈 1명 충원이 갈수록 어려움 | SSOT/브리프 |

→ 논리: 회사가 VisaDesk로 근로자를 등록하면 근로자는 Moneeback으로 **자기 돈(환급)을 찾는다**. "받을 돈을 챙겨주는 회사"는 근로자가 떠날 이유 하나를 지운다(근속↑ 인과는 〔가설〕 — PoC에서 근로자 만족·잔존으로 프록시 측정). 엑셀·더존·행정사 중 누구도 **근로자에게 돈을 찾아주지 못한다**. 이것이 "준법 SaaS"가 아니라 **"외국인 인력 유지 OS(준법+복지)"** 리포지셔닝의 근거다.

#### A-2. 정직한 한정 (격파가 아니라 한정이 맞는 경우)

4조건이 모두 약한 고객 — 외국인 ≤5명·단일 비자·이탈/단속/점검 무경험·증원 계획 없음 — 에게는 **엑셀이 실제로 합리적 선택**이다. 이 고객에게 팔려는 시도는 CAC 낭비 + 조기 해지 + 평판 비용이다. WTP는 보편 명제가 아니라 **조건부 명제**이고, GTM의 일은 조건을 가진 고객을 찾는 것(자가진단=조건 스크리너)이다.

### E. 가격 실험 설계 (PoC 8주 내 실측)

> 대원칙(04 정합): **정가(29/89/169)는 SSOT 고정 — PoC 본판(15사 트랙)에서는 가격을 건드리지 않는다.** 가격 변주 실험은 본판을 오염시키지 않도록 **별도 코호트(자가진단 유입 중 anti-ICP 인접·5~10명 소형사)**에서만 실행. 모든 전환율 기준은 〔실험안·가정〕.

|---|---|---|

## ③ So-what

1. **창업자 의심의 답은 "격파"가 아니라 "조건부 격파"다.** 엑셀은 N≤5에서 실제로 이긴다 — 이를 인정하는 순간 N≥10에서의 우리 논리(시간+기대손실+증발)가 과장 없이 성립하고, 자가진단이 그 조건을 스크리닝하는 도구가 된다. "모두에게 필요하다"는 주장보다 "10명부터 필요하다"는 주장이 더 많이 판다.
2. **WTP의 차별화 축은 준법이 아니라 유지다.** 조건 1~3(처벌·임계점·제도)은 언젠가 경쟁자도 말할 수 있지만, 조건 4(환급 복지→근속)는 Moneeback을 가진 우리만 말할 수 있다 — B2C 4사 조사에서 환급 트랜잭션 보유사 0(’26.6) 재확인. 리포지셔닝("인력 유지 OS")은 카피 변경이 아니라 해자 위에 메시지를 올리는 일이다.
3. **구독료의 내부 역할(진지함 필터·데이터 입구) 때문에 freemium은 기본값이 될 수 없다.** 무료화는 활성화 붕괴(07 R1)와 플라이휠 단절 리스크 — 실험 1의 decision rule을 통과할 때만 제한적으로 연다.
4. **보수 시나리오에서 ROI가 0.8배라는 사실을 영업이 먼저 말해야 한다.** 그것이 "이탈·단속 경험 있는 30명 사업장"이라는 S등급 타겟팅의 근거이자, 영세 사업자 시장에서 신뢰(= 전환율)를 만드는 비용이다.
